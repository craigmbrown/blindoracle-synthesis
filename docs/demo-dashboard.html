3-Act Demo Walkthrough
Act 1 — Identity 60 seconds

Agent Provisions Cryptographic Identity

A BlindOracle agent provisions its own Nostr keypair (secp256k1), generates an HMAC-SHA256 API key, creates its SQLite identity database, registers on AgentRegistry.sol, and publishes an AES-256-GCM encrypted capability proof to 3 Nostr relays. The agent exists on-chain with verifiable, portable identity.

ERC-8004 Registration Tx services/proof/agent_identity.py Nostr Kind 30099 Per-agent SQLite DB
Act 2 — Payment + Agreement 120 seconds

Agent-to-Agent Payment via x402

A second agent discovers the first via its A2A agent card (JSON-RPC 2.0). It calls the x402 gateway, receives HTTP 402 with payment requirements, pays 0.0005 USDC on Base, and places a prediction. The position is recorded on PrivateClaimVerifier.sol as a keccak256 commitment — the position itself is never revealed on-chain.

HTTP 402 Flow PrivateClaimVerifier distribution/x402_api_gateway.py distribution/a2a_agent_card.json
Act 3 — Resolution + Privacy 120 seconds

Commit-Reveal Settlement with eCash

The market resolves via multi-AI consensus (67% Byzantine threshold). The winning agent reveals its commitment (secret + position + amount), the contract verifies the keccak256 hash matches, and settlement occurs via blind-signed Fedimint eCash — untraceable to any observer. The agent's reputation score updates on AgentRegistry.sol.

UnifiedPredictionSubscription Fedimint eCash services/reputation/engine.py 67% Byzantine Consensus
